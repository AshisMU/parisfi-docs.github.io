Troubleshooting & Tuning

Running robust AI infrastructure locally requires balancing speed against Context Limits.

Understanding Map-Reduce & Token Limits

Every Local LLM has a "Context Window" (e.g., Llama-3 often defaults to 8,192 tokens). If you feed a 500-page document into the AI all at once, the context window will overflow, causing the AI to hallucinate or return an error.

To safely process massive documents, Parsifi utilizes an automated Map-Reduce Engine.

  • The Trigger: If a document's token count exceeds the threshold in settings, Map-Reduce activates automatically.
  • Map Phase: Parsifi chunks the document (e.g., 2,000 tokens per chunk) and asks the LLM the same prompt for every chunk individually.
  • Reduce Phase: Parsifi takes all the individual answers and submits a final prompt to the LLM to aggregate them into one cohesive response.
The Cost of Map-Reduce: While it guarantees the entire document is read, Map-Reduce is significantly slower because the LLM must process the prompt 10x or 20x more times than a single-pass document. Use Targeted Content Extraction (Smart Chunking keywords) to avoid Map-Reduce when speed is paramount.

Truncation Warnings

If you use Collection Tags (e.g., {{children.content}}) to combine 50 different documents into a single prompt, Parsifi will attempt to prevent your LLM from crashing by implementing a hard-stop limit (100,000 characters). If you see a warning that your text was TRUNCATED:

  1. Stop using the raw `{{children.content}}` tag.
  2. Use the `{{children.analysis}}` tag instead. By summarizing the children first and then accumulating the summaries, you compress 100,000 characters down into 5,000 highly potent characters that fit effortlessly in the context window.
  3. Use Cross-Document Referencing: If you only need data from one or two specific external files, reference them directly using {{ext_DocumentName.step_StepName}} to surgicaly inject only the necessary data points rather than dumping entire collections into the prompt.

Local vs "Cloud" Speeds

If Parsifi feels slow, it is strictly a limitation of the Local LLM running on your computer's hardware. You cannot compare the generation speed of an 8B model running on a consumer GPU to GPT-4 running on thousands of enterprise H100 arrays.

To Improve Local Speed:

  • Switch to a smaller model in LM Studio (e.g., Qwen 1.5B or standard Llama 8B).
  • Lower the quantization level (e.g., Q4_K_M instead of Q8_0) to save VRAM and improve memory bandwidth speeds.
  • Offload 100% of layers to GPU inside LM Studio. If your model spills over into CPU RAM, generation speed drops from 40 tokens/sec to 2 tokens/sec.

Document Failed to Process

If a document throws an error during the VLM parsing stage:

  1. Rethink the pipeline: Does this document need the vision model? Can you just extract text via Standard OCR?
  2. If the document is a massive PDF (1,000+ pages), the VLM will take hours to process locally. Try splitting the PDF before uploading.
