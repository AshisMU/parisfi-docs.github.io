Pipeline Version History & Restore

Every time you save a pipeline canvas, Parsifi creates an immutable version snapshot. Learn how to browse the history, compare changes, and safely restore a previous graph configuration.

Prerequisites: You need an existing Orchestration Pipeline. See Tutorial 07 or Tutorial 08. This tutorial is most useful if you have already made several canvas saves and want to track or revert changes.

How Versioning Works

When you first create a pipeline (via the Smart Wizard or by saving a new canvas), Parsifi automatically records Version 1. Every subsequent time you click Save Canvas, a new version row is created. The pipeline's "live" state is always the most recent version, but all previous versions are preserved indefinitely.

Step-by-Step

1

Open the Pipeline in the Graph View

Navigate to Orchestration → Data Flow Graph, and select your pipeline from the dropdown. The canvas loads and the toolbar shows the ðŸ• History button.

2

Open the History Panel

Click the ðŸ• History button. The right inspector panel will display the full version timeline in descending order (newest at the top). Each entry shows:

  • Version number — e.g. "v3", with a "Current" badge on the latest one.
  • Change note — either a custom message (future feature) or an auto-generated label like "Canvas save #2".
  • Timestamp — the date and time the save was made.
3

Restore a Previous Version

Click the Restore button next to any archived version. Parsifi will load its exact nodes and edges back onto the canvas. A toast notification will confirm: "Restored to v1. Click Save Canvas to make it permanent."

Restoring only loads the old state onto the canvas — it does not automatically save. You must review the restored graph first, then click Save Canvas to commit it as a new version.
4

Review and Save the Restored Canvas

After restoring, inspect the graph to confirm this is the correct state. When ready, click Edit Canvas to enter edit mode, and then click Save Canvas. This creates a new version (e.g. "v4 — Canvas save #4") which is now the new "Current" version. The full history from v1 to v4 is preserved.

5

Run the Pipeline

After a successful restore and save, running the pipeline will use the newly restored graph configuration. Navigate to the My Pipelines tab and click Run, or use the Play button on the graph tab toolbar.

Best Practices

  • Save deliberately: Each canvas save creates a version. Avoid mindless saves — save when you reach a meaningful, tested milestone.
  • Before a big change, verify the current state: Open the History panel to confirm what "current" looks like before making major structural edits.
  • Compliance audit trail: Version timestamps serve as a timestamped record proving which logic was active at any point in time — valuable if a regulatory question arises about a document generated months ago.
