Tutorial 9: Mastering the Variable Register

Learn how to trace data lineage across your entire enterprise deployment and debug missing data links using the Master Variable Register.

1

Identify an Orphaned Output

Create a new Template and upload a dummy Word document containing the text: {{phantom_variable}}.

Click Inspect Required Fields on this template to register the variable.

2

Navigate to the Master Register

In the main app sidebar, click Master Variables under the Orchestration heading.

This is the global dictionary tracking every known variable field across your tenant.

3

Inspect Lineage

Search for `phantom_variable` in the list and click its info icon to view details.

You will see a modal split into two halves: Extracted By (Tools) and Required By (Templates).

Notice that your new template is listed under "Required By", but the "Extracted By" section is completely empty.

4

Complete the Circuit

To fix the orphaned output, you need an Extractor that provides this data.

Create a new Agent Tool with a JSON Schema containing a string property precisely named `phantom_variable`.

Return to the Master Variable Register and inspect `phantom_variable` again.

Verify Success: The variable lineage modal will now show the new Agent Tool listed as the source ("Extracted By") and your Template as the destination ("Required By"). Because this logical link exists in the Register, any Orchestration Pipeline containing both will successfully map data between them with zero manual configuration.
