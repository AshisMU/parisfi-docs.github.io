Understanding the Data Lineage Map

Use the visual data overlay on the pipeline canvas to verify which variables are correctly mapped, which are missing, and which have naming conflicts — all before running anything.

Prerequisites: You should have at least one Orchestration Pipeline with an Extractor node (linked to an Agent Tool with a JSON schema) and a Generator node (linked to a Template). See Tutorial 08 if you need to build one first.

What Is the Data Lineage Map?

The Data Lineage Map is a visual overlay you can toggle on the Pipeline Canvas. Instead of showing just the execution order (Node A â†’ Node B), it shows you the actual data variables flowing between nodes — which variables are produced by your Extractors, and which are consumed by your Generator Templates.

Step-by-Step

1. Open the Pipeline Canvas

Navigate to Orchestration → Data Flow Graph. Select a pipeline from the dropdown. The canvas will load with your nodes and edges in execution-order view.

Once the graph is loaded, the toolbar will show the ðŸ‘ Data Map button in teal.

2. Click "Data Map" to Activate the Overlay

Click the ðŸ‘ Data Map button. Parsifi will:

  • Fetch the JSON schema for every Extractor node's linked Agent Tool to get its produced variable names.
  • Fetch the required_fields list for every Generator node's linked Template to get its consumed variable names.
  • Cross-reference the two lists and inject variable ellipse nodes and dashed edges onto the canvas.

A toast notification will appear summarising: "Data Map: 5 mapped Â· 1 unmet âš ".

3. Inspect the Legend in the Side Panel

The right inspector panel switches to the Data Map Legend, which explains each colour state. You can scroll the canvas to trace the dashed edges between Extractor â†’ Variable â†’ Generator.

4. Resolve Red (Unmet) Variables

A red variable node means your template requires a field that nothing produces. Common fixes:

  • Add it to an Extractor's schema: Open the Agent Tool linked to your Extractor node and add the missing property to its JSON Output Schema.
  • Add a new Extractor node: If the missing data comes from a different document, add another Extractor node and link it to a tool that does produce that field.
  • Remove it from the template: If the field is optional, simply remove the placeholder from your DOCX/PDF template.

5. Resolve Orange (Collision) Variables

An orange variable means two Extractors produce the same JSON key. Whichever Extractor runs last will win. To resolve:

  1. Open the Agent Tool linked to one of the colliding Extractors.
  2. Rename the conflicting property in its JSON Output Schema (e.g. rename total_amount to invoice_total).
  3. Update the template placeholder to match ({{invoice_total}}).
  4. Refresh the Data Map by clicking Hide Map then Data Map again.

6. Hide the Map and Run

Once all variables are teal (mapped), click Hide Map to return to the normal execution canvas. You can now be confident that all data dependencies are satisfied and click Run to execute the pipeline.

Use the Data Map as your pre-flight check every time you add a new Generator or modify a Tool schema. Checking it takes 5 seconds and prevents blank fields in generated documents.
