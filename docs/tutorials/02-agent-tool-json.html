Tutorial 2: Building Strict JSON Tools

Move beyond simple chat by creating an Agent Tool that forces the LLM to return data exactly according to a JSON Schema.

1

Create a JSON Schema

Navigate to Advanced > JSON Schemas. Click + New Schema.

Name it "Invoice Schema". Add two properties: invoice_number (type: string) and total_amount (type: number).

Save the schema.

2

Create an Agent Tool

Navigate to Advanced > Tool Factory. Click + New Tool.

Name it "Invoice Extractor".

In the System Prompt, write: "You are a strict data extractor. Extract the invoice number and total amount from the document."

Under Output Schema, select the "Invoice Schema" you just created.

Ensure Strict Mode is toggled ON to guarantee schema compliance.

Save the tool.

3

Test the Tool

Navigate to the Document Viewer and open an invoice document.

In the right-hand panel, switch from the Analysis tab to the Agent Tools tab.

Select your "Invoice Extractor" tool from the dropdown and click Run Tool.

Verify Success: The output will not be a chatty paragraph. It will be a perfectly formatted JSON object containing exactly the keys invoice_number and total_amount with their corresponding values extracted from the document.
