Tutorial 8: Multi-Input Pipelines (DAGs)

Move away from linear workflows and build a true Directed Acyclic Graph (DAG) that waits for multiple distinct document uploads before generating a combined report.

1

Prepare the Component Parts

Ensure you have two Agent Tools ready: "Purchase Order Extractor" (extracts `po_value`) and "Invoice Extractor" (extracts `inv_value`).

Ensure you have a Template ready: A DOCX template that requires both `{{po_value}}` and `{{inv_value}}` side-by-side.

2

Configure Pipeline Inputs

Create a new Pipeline using the Smart Wizard.

In the "Input Slots" section, define TWO document requirement slots: call the first "Purchase Order" and the second "Vendor Invoice".

Select both of your Extractor tools and your Generator template, then click Create.

3

Navigate the Upload Modal

Click Run Pipeline on the Data Flow Graph canvas.

Notice the upload modal: unlike Workspaces which just ask for a single file, the pipeline execution modal explicitly demands files for both the "Purchase Order" slot and the "Vendor Invoice" slot.

You cannot launch the pipeline until both slots are satisfied.

4

Observe Parallel Execution

Upload a dummy document into each slot and launch.

Verify Success: Observe the canvas behaviour during execution. Both extractor nodes will spin concurrently (parallel execution) since they depend on different inputs. The final Generator node will wait for both upstream extractors to finish before producing the final merged Word document containing structured data sourced from two separate physical files.
