Placeholder Tags & Variables

A comprehensive reference for all dynamic placeholder tags and variables available in Parsifi's prompts, tools, and workspaces.

Parsifi utilizes a powerful VariableResolver to inject dynamic data into your instructions. Tags are evaluated using the {{tag_name}} syntax.

1. Document Level Content Tags

Tag Implementation Scenario
{{document.content}} VariableResolver.resolve Injects the full markdown text of the current document.
{{document.name}} resolve_system_tag The original filename or display name of the document.
{{document.id}} resolve_system_tag The unique database ID for the document.
{{summary}} System Alias Injects the latest analysis summary (JSON) into the prompt.

2. Workspace Chaining (Step Outputs)

Workspaces allow you to pass data between steps using the {{StepName.output.key}} pattern.

Tag Result
{{Summarize.output.total_tax}} Extracts the 'total_tax' field from a previous step named 'Summarize'.
{{Step1.output.raw_response}} Injects the entire text response from the first step.

3. Collection & Multi-Document Tags

When running a prompt on a collection "Master", you can query all its children.

Tag Description
{{children.content}} Merges the content of all child documents into one context.
{{children.analysis}} Aggregates the summaries of all child documents.
{{collection.document_count}} The total number of files in the collection.

4. External Document Referencing

Query documents that aren't even in your current project using {{ext_DocName.step_StepName}}.

Example: {{ext_Policy_2024.pdf.step_Analysis.output.limit}} pulls the 'limit' field from the 'Analysis' result of 'Policy_2024.pdf'.

5. Repeat Step — Loop Variable Tags

When a Repeat step runs an inner Agent Tool once per list item, the current item is injected into the inner tool's prompt as a special tag.

Tag Description
{{current_item}} Default loop variable. Resolves to the current list item (string, number, or serialized object) for this iteration of the Repeat step.
{{your_custom_name}} If you set a custom Loop Variable Name in the Repeat wizard's Advanced section, use that name instead. For example, if you set the loop variable to current_director, use {{current_director}} in the inner tool's prompt.
How it works: Each loop iteration injects the current item as a custom variable into the inner step before execution. The inner tool's prompt is re-rendered fresh for every item in the list, so each run is fully independent. See Pipelines & Workspaces for a step-by-step guide to configuring Repeat steps.
