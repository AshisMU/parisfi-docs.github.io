Professional Workflows & Best Practices

Learn how to combine Parsifi's distinct features into powerful, end-to-end automated pipelines for common professional scenarios.

Use Case 1: Automated Invoice Processing

Goal: Extract vendor names, line items, and totals from scanned invoices, outputting them into a standardized Excel sheet without manual intervention.

  1. Setup Taxonomy: Create an "Invoice" category with the keyword "Total Amount Due".
  2. Create Tool: Build an Agent Tool named "Extract Invoice Data". Write a prompt emphasizing JSON Intent (e.g., extract VendorName and Total). Upload an Excel template that maps {{VendorName}} to cell A1.
  3. Build Workspace: Create a Workspace named "Invoice Pipeline".
    • Step 1: Auto-Categorize
    • Step 2 (Agent Tool): Run "Extract Invoice Data"
    • Step 3 (Branch 🔀): Route on category — YES path if "Invoice", NO path otherwise
  4. Execution: On the Dashboard, drag 50 scanned PDFs into the upload modal, select "Vision VLM Processing" (since they are scans), and select the "Invoice Pipeline" workspace. Press Start.
  5. Result: Parsifi will parse all 50 images, verify which ones are invoices, extract the data, and generate Excel sheets only for confirmed invoices.

Use Case 2: Corporate Filing — Per-Director KYC

Goal: Extract a list of directors from a corporate filing and run a KYC compliance check on each one automatically.

  1. Create Extraction Tool: Build an Agent Tool named "Extract Directors" with a JSON Schema that includes a directors array field (each item an object with name and id_number fields).
  2. Create KYC Tool: Build a second Agent Tool named "Director KYC". In the prompt, reference the current director via {{current_director}}.
  3. Build Workspace: Create a Workspace named "Corporate KYC Pipeline".
    • Step 1: Auto-Categorize
    • Step 2 (Agent Tool): Extract Directors
    • Step 3 (Branch 🔀): Route on entity type — YES path if "Corporate", NO path for individuals
    • Step 4 (Repeat 🔁): For each director in the directors list, run "Director KYC"
  4. Run: Upload a corporate filing document and select the workspace.
  5. Result: Parsifi extracts all directors, routes the document as corporate, and runs the KYC tool once per director — automatically. Results are stored as Director KYC_item_0, Director KYC_item_1, etc.

Use Case 3: Resume Screening & Candidate Comparison

Goal: Summarize 100 applicant resumes and aggregate the best candidates for a specific job description.

  1. Upload Context: Go to the Knowledge Base. Upload the official "Data Scientist Job Description.txt".
  2. Setup Prompt: Write a prompt: Review this resume. Compare it strictly to the attached Job Description. Score the candidate out of 10. Include the {{context}} tag.
  3. Execute: Go to Dashboard. Upload all 100 PDF resumes using standard OCR (since text is easily readable). Select all 100 documents. Click Run Workspace from the bulk action bar, select your prompt, and select the Job Description from the Context Dropdown.
  4. Aggregate: Once done, add the top 10 candidates to a Collection named "Top Candidates". Run a final Workspace command on the Collection using the {{children.analysis}} tag: "Summarize the pros and cons of these top 10 candidates into a single executive summary."

Use Case 4: Contract Variance Analysis

Goal: Detect clauses in a 200-page vendor contract that violate your company's standard policy.

  1. Upload Policy: Upload your company's "Vendor Policy Guidelines" to the Knowledge Base.
  2. Smart Chunking: Upload the 200-page contract using standard OCR. Opening it in the split-viewer, enter keywords in the Smart Chunking bar: liability, indemnification, warranty.
  3. Execute: Parsifi will slice the 200-page contract down to only the relevant pages. Run your "Find Policy Violations" prompt using the Vendor Policy as Context. The result is instant and highly accurate because the LLM wasn't distracted by 180 pages of definitions.
Tip — Order your steps carefully: Branch and Repeat steps must come after the steps whose output they reference. Use the ▲ ▼ reorder buttons in the Workspace editor to adjust step order without deleting and recreating them.
