JSON Schemas

JSON Schemas are the key to turning AI output from free-form text into reliable, structured data — ready to populate reports, spreadsheets, and downstream pipeline steps.

What is a JSON Schema?

A JSON Schema is a simple description of what fields you expect the AI to extract. When you define one on a Tool or Prompt, Parsifi instructs the AI to return exactly those fields — nothing more, nothing less. Think of it as a form you hand to the AI and say: "Fill in these blanks."

No schema? That's fine too. Without a schema, the AI returns a narrative summary — useful for reading, but harder to feed into a spreadsheet or a downstream step.

When to Use a Schema

Goal Use a schema? Why
Read a summary of a document No Free-form text is fine for reading
Populate an Excel report Yes Each schema field maps to a cell
Chain step output into the next step Yes Schema fields appear in the variable helper
Extract a list of items (directors, line items) Yes — use an array field Required for Repeat steps (Phase 1)
Prevent the AI from guessing/inventing data Yes + Strict Mode Schema + Strict = verbatim extraction only

Three Schema Types

1. Flat Schema — simple fields

Use this for straightforward documents where each piece of information is a single value.

Example: Invoice extraction

{
  "type": "object",
  "properties": {
    "invoice_number": { "type": "string" },
    "vendor_name":    { "type": "string" },
    "invoice_date":   { "type": "string" },
    "total_amount":   { "type": "number" }
  }
}

The AI will return:

{
  "invoice_number": "INV-2024-001",
  "vendor_name":    "Acme Corp",
  "invoice_date":   "2024-03-14",
  "total_amount":   1500.00
}

2. Nested Schema — grouped fields

Use this when some fields logically belong together, like a registered agent's name and address.

Example: Company registration

{
  "type": "object",
  "properties": {
    "company_name":  { "type": "string" },
    "entity_type":   { "type": "string" },
    "registered_agent": {
      "type": "object",
      "properties": {
        "name":    { "type": "string" },
        "address": { "type": "string" }
      }
    }
  }
}

In the variable helper, registered_agent appears as a single option. To access a sub-field, reference it in your prompt by name (e.g. "Extract the registered agent's name separately").

3. Array Schema — lists of items

Use this when a document contains a repeating list — like multiple directors, line items, or clauses. Array fields are marked (list) in the variable helper and are required for Repeat steps.

Example: Board of directors

{
  "type": "object",
  "properties": {
    "company_name": { "type": "string" },
    "directors": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "name":      { "type": "string" },
          "role":      { "type": "string" },
          "id_number": { "type": "string" }
        }
      }
    }
  }
}
Arrays and Repeat Steps: A Repeat step loops over an array field. For the loop to work, the feeding tool must have a JSON schema with the target field typed as "type": "array". Without this, the AI returns text and the loop has nothing to iterate over.

How to Add a Schema in the Tool Editor

  1. Go to Tools in the top navigation.
  2. Select or create a tool.
  3. Scroll to the JSON Schema [Optional] section.
  4. Either:
    • Type or paste your schema directly, or
    • Click + String Field, + Number Field, or + List Field to scaffold one quickly.
  5. The editor shows ✓ Valid JSON or ✗ Invalid JSON as you type.
  6. Once valid, Fields available: lists all top-level field names extracted from the schema — these are the fields that will appear in the workspace variable helper.
  7. Click Save.
Quick-insert buttons: Start from scratch? Click + String Field to get a complete valid schema with one string field. Add more fields by clicking again — Parsifi merges them into the existing schema automatically.

Schema + Strict Mode

Enabling Strict Mode alongside a JSON Schema gives you the highest reliability for data extraction. The combination tells the AI:

"Return exactly these fields. If a value is not explicitly stated in the document, return null — do not guess."

Use this for regulated documents, legal filings, or any extraction where invented data is unacceptable.

Common Mistakes

Mistake Fix
Schema saved but variable helper still shows "No schema" Make sure the tool is saved first, then re-open/select it to refresh the helper
JSON schema shows ✗ Invalid Check for trailing commas, missing quotes around field names, or mismatched braces
Array field not showing in Repeat step picker The field must be typed as "type": "array" — not just a string containing a list
AI returns extra fields not in schema Enable Strict Mode to constrain output to schema fields only
