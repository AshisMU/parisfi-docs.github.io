Template Fill Engine

Automate data entry by securely binding extracted AI structured data into your physical company templates.

Overview

Parsifi can fill native document templates across dual pathways without losing formatting or triggering complex intermediate abstractions. The execution leverages a specialized fill_template step within a Workspace.

Direct Delivery: Outputs are strictly generated in their native formats. A filled XLSX will download as an XLSX; an AcroForm PDF will download as a PDF. No awkward conversions.

Mode 1: AcroForm & Static PDFs

Use Mode 1 when dealing with pre-authored PDF forms or legacy static visual templates.

Uploading Native Forms

Just upload the AcroForm PDF to Parsifi's Knowledge Base. The underlying template engine automatically detects the input layout and scans for named fields (e.g. "Client_Name").

The Path 2 In-app Visual Editor

If you upload a flat, static PDF, Parsifi ships with an integrated Form Field Editor. Open the document in the Knowledge Base viewer, activate the Edit Fields toggle, and draw AcroForm widget rectangles directly over the physical blanks in the document. Name these fields accurately, and the system permanently attaches the spatial map to the document.

Mode 2: Network Folder Templates

Use Mode 2 for Word Documents, Excel Spreadsheets, and PowerPoint Presentations.

Instead of manually uploading each varying version of your firm's `.docx` or `.xlsx` files, place them in a designated folder on your machine.

  1. Tag physical areas in your documents using the `{{placeholder_name}}` syntax.
  2. Configure the Template Folder Path inside Settings.
  3. Click Scan Folder in the UI. Parsifi idempotently syncs the folder content into the Knowledge Base without storing copies in the database.

Linking AI Output to Templates

In the Workspace editor, add a Fill Template step directly after an Agent Tool extraction. The intelligent step wizard allows you to cross-reference AI JSON keys directly into the mapped template fields.
