Variable Register

The Variable Register is the central nervous system connecting your Agent Tools, Templates, and Pipelines. It ensures data flows seamlessly without manual mapping.

The Problem It Solves

In traditional automation, if an Agent Tool extracts `{"Company_Name": "Acme Corp"}`, but your template expects `{{business_name}}`, the pipeline will fail unless you write custom code to map the fields. The Variable Register eliminates this by maintaining a central dictionary of known fields.

How it Works

When an Orchestration Pipeline runs, the data flow looks like this:

  1. Extraction: An Extractor node runs an Agent Tool. The JSON output items are placed directly into the pipeline's memory context using their JSON keys as the variable names.
  2. Generation: A Generator node loads a Template. It scans the template for placeholders and looks them up in the memory context.
Zero-Configuration Mapping: As long as your Agent Tool's JSON Schema keys match the `{{placeholder}}` names in your Template, data will map automatically. You never have to manually wire fields together.

Syntax: Single vs Double Braces

Syntax Used In Description
{single_brace} Agent Tool Prompts Tells the AI where to inject system memory or pipeline inputs into the prompt before sending it.
{{double_brace}} Templates (DOCX, HTML) Tells the Generator node where to inject extracted data into the final document.

Auto-Registration

You rarely need to create variables manually. Parsifi uses Auto-Registration:

  • When you save an Agent Tool with `{fields}`, they are automatically registered as inputs.
  • When you upload a Template with `{{fields}}`, they are automatically registered as outputs.
  • When you define a JSON Schema for an Agent Tool, the properties are registered as pipeline variables.

Managing the Register

Navigate to Orchestration > Master Variables in the main app to view the Register. Here you can:

  • See every variable used across your entire tenant.
  • Click on a variable to see exactly which Tools use it as an input, and which Templates use it as an output.
  • Spot Orphaned Variables — e.g. a Template requires `{{contract_value}}` but no Agent Tool is configured to extract it.

System Variables

In addition to your own custom data, certain system variables are always available in Agent Tool prompts using {single_brace} syntax:

Variable Available In Description
{document_text} Agent Tools The full OCR text of the document currently being processed.
{current_item} Workspace Repeat Nodes The current item being iterated over in a Repeat step loop.
{input_1}, {input_2}... Orchestration Pipelines References the slot data uploaded by the user when launching the pipeline.
