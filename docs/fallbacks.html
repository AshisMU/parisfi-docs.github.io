System Fallbacks

Learn about the built-in system fallbacks designed to ensure continuous operation and seamless error handling across Parsifi.

1. Prompts with Fallbacks

The "Default" Prompt

This prompt serves as the ultimate fallback set of AI instructions ("You are an expert document analyst...").

If a user provides an invalid prompt key or doesn't specify one, the system falls back to this default set of instructions to ensure an analysis is still generated.

Prompt Recommendations UI

The frontend uses a fallback hierarchy when suggesting prompts to the user. If a document-specific prompt key isn't available, it falls back to category-based recommendations. As a final resort, it defaults to the first available prompt or the Default prompt.

2. Artifacts & Core System Components

Document Chunking & Splitting

The system actively attempts to use hierarchical chunking based on document structure (e.g., structural JSON outputs). If no structural layout is found or the hierarchical chunking fails, the system safely triggers a fallback split method using standard text-length chunking.

LLM Output Parsing

If the LLM Agent exhausts all retry attempts while trying to parse valid structured outputs, the system avoids crashing by triggering a fallback. It purposely returns a valid, basic JSON format to prevent an "Agent Parse Error" and ensure the pipeline continues running smoothly.

Taxonomy Loading

When initializing, the service attempts to load a taxonomy map from taxonomy.json. If this file fails to load or is unreadable, it implements a fallback by using a hardcoded, default taxonomy dictionary.

Token Counting for Processing

The processing pipeline computes chunk limits using the tiktoken tokenizer. If the tokenizer fails to load or calculate length properly, it includes a fallback mechanism to use standard character counting instead.

Pydantic Data Validation

Includes a specific error validation fallback to handle formatting anomalies—for instance, if the pages configuration is mistakenly parsed as a list, the system catches it and corrects it on the fly.

Workspace/Collection Data Structures

The dashboard logic includes a data fallback to seamlessly support both legacy array formats and the newer object structures. This ensures backward compatibility for older workspaces without breaking the UI.
