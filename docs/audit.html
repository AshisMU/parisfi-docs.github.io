Audit Log

Gain absolute administrative oversight with Parsifi's tamper-evident, comprehensive history tracker.

Ensuring Compliance

For financial, legal, and operational institutions running Parsifi on local servers, data governance is paramount. The Audit Log continuously captures all meaningful state changes in the platform.

What is Tracked?

The system tracks mutating events. Non-mutating read events (like opening a document or viewing a setting) are omitted to minimize dashboard noise.

  • Authentication: All user logins and logouts.
  • Document Lifecycle: Uploads, hard deletions, and image preprocessing operations (rotate, erase tool, contrast).
  • Analysis & Extractions: Triggers for parsing, pipeline executions, step definitions used, and direct user modifications to extracted results.
  • Agent Tools & Prompts: Authoring, deleting, or altering LLM instructions and schema configurations.
  • System Mechanics: Application setting changes, model updates, and knowledge-base template directory scans.
Entity Relationships: Logs capture the user context recursively, documenting the exact timestamp, actor IP address, action signature, and affected filename natively.

Using the Viewer

To access the viewer, click on Audit Log at the bottom of the main application sidebar navigation.

You can rapidly filter past activity using the top criteria bar and click on any individual row to expand a highly detailed JSON blob view that encapsulates deeper operational context or the exact configurations altered during the event.
